## 구현

- 충분하지만 지나치치 않은 수준으로 해결책을 개발합니다.
- 추상화보다 선언적 접근을 우선시합니다.
- 약간의 장황함은 영리한 코드보다 낫습니다. 사람들의 기여 능력을 저하시키는 구현은 피하세요.
- 추상화와 자동화를 통해 반복을 줄입니다.
- 유연성을 제공하되, 합의된 경계 내에서 제공합니다.
- 사람들이 규칙을 어길 것이라고 가정하고, 이를 안전하게 수행할 수 있는 방법을 제공합니다.
