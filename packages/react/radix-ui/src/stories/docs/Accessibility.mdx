# 접근성(Accessibility)

![접근성](https://img.shields.io/github/actions/workflow/status/leboncoin/spark-web/playwright-a11y.yml?label=WCAG%202.1%20AA)

## 소개

접근성은 **시각, 청각, 운동 및 인지 장애와 같은 장애가 있는 사람을 포함하여 모든 사람이 애플리케이션을 사용할 수 있도록 하는 기능**입니다.

**DS 디자인 시스템**은 스크린 리더 및 키보드 내비게이션에 대한 지원과 같은 접근성의 주요 측면을 다루며, 사용자 경험을 저해하지 않고 디자인 프로세스에 원활하게 접근성을 통합할 수 있도록 합니다.

장애가 있는 사용자에게 도움이 될 뿐만 아니라, 접근성 기능은 모든 사용자의 경험을 향상시킬 수 있습니다. 예를 들어, 많은 사용자가 마우스나 터치스크린 대신 키보드를 사용하여 애플리케이션과 상호 작용하는 것이 더 빠르고 효율적이라고 느낄 수 있습니다.

## 웹 콘텐츠 접근성 지침(WCAG)

접근성은 시맨틱, [WAI-ARIA](https://www.w3.org/TR/wai-aria-1.2) 사양 및 동작 구현에 기반합니다.

명확한 지침을 제공하기 위해 W3C는 패턴 및 실제 사례를 설명하는 [ARIA 작성 관행 가이드](https://www.w3.org/WAI/ARIA/apg/)를 발행합니다. 또한 예상되는 키보드 상호 작용과 보조 기술을 위한 필요한 역할 및 상태를 정의합니다. 마지막으로 포괄적인 [WCAG 2.1](https://www.w3.org/TR/WCAG21) 사양은 더 자세한 지침을 제공합니다.

**DSDS** 컴포넌트는 **WCAG 2.1 레벨 AA** 지침 및 일반적인 접근성 모범 사례를 준수하여 **모든 사용자에게 포괄적이고 사용자 친화적인 경험**을 제공합니다.

<Callout kind="warning">
  접근성은 구현 과정에서 쉽게 손상될 수 있으므로 모든 사용자에게 포괄적이고 사용자 친화적인 경험을 유지하도록 주의해야 합니다.
</Callout>
